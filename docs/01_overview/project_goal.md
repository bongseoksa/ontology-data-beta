# Phase 1: 프로젝트 개요 정의

---

## 1. 문제 정의 (Problem Statement)

### 1.1 현재 상황
중소기업 대표들이 소프트웨어 프로젝트 가격을 산정할 때 직면하는 문제:
- 사업계획서 기반 견적 산정의 어려움
- 전문 인력 부재로 인한 비합리적 가격 책정
- 시장 상황과 리스크를 반영한 전략적 가격 결정의 한계

### 1.2 해결하고자 하는 문제
**비정형 사업계획서를 입력받아 구조화된 프로젝트 모델로 변환하고,
다중 전문가 관점의 분석을 거쳐 합리적인 가격을 자동 산정하는 시스템 부재**

### 1.3 시스템의 본질
이 시스템은 단순 견적 계산기가 아닌:
- **문서 이해** (Document Understanding)
- **구조화** (Structuring)
- **다중 전문가 판단** (Multi-Expert Reasoning)
- **계산 엔진** (Calculation Engine)

이 결합된 **AI 기반 의사결정 자동화 시스템**이다.

---

## 2. 핵심 목표 (Core Objectives)

### 2.1 1차 목표: 구조화
```
사업계획서 (비정형 문서) → 구조화된 프로젝트 모델 (정형 데이터)
```

### 2.2 2차 목표: 원가 계산
```
구조화된 프로젝트 모델 → Base Cost 산출
```

### 2.3 3차 목표: 전략적 가격 산정
```
Base Cost → Final Price (마진, 리스크, 시장 전략 반영)
```

### 2.4 4차 목표: 의사결정 지원
```
다중 전문가 시뮬레이션 기반 판단 근거 제공
```

---

## 3. 최종 산출물 (Deliverables)

### 3.1 주요 산출물

| 산출물 | 설명 |
|--------|------|
| **최종 가격** | 전략적 조정이 반영된 프로젝트 견적 금액 |
| **가격 산정 근거** | 각 비용 항목별 산출 내역 |
| **리스크 분석** | 식별된 리스크 및 프리미엄 적용 근거 |
| **전략 가중치 설명** | 시장 상황 및 경쟁 분석 기반 조정 근거 |

### 3.2 출력 형태
- 최종 가격 리포트 (PDF/웹)
- 구조화된 JSON 데이터 (시스템 연동용)
- 대화형 설명 (사용자 질의 응답)

---

## 4. 타겟 사용자 (Target Users)

### 4.1 Primary
- **중소기업 대표**: 소프트웨어 프로젝트 발주 또는 수주 시 가격 산정 필요
- **스타트업 창업자**: 외주 개발 또는 자체 개발 비용 추정 필요

### 4.2 Secondary
- **SI 영업 담당자**: 입찰 가격 산정 시 참고
- **PM/기획자**: 프로젝트 예산 수립 시 활용

---

## 5. 핵심 페르소나 (Key Persona)

> "기술 자랑이 아니라 **의사결정 구조를 어떻게 설계했는가**를 보여줘야 한다"

### 5.1 차별화 포인트
- **파운더리 온톨로지 기반**: 도메인 지식의 체계적 구조화
- **다중 전문가 시뮬레이션**: 기술/사업/경제 관점의 균형 잡힌 분석
- **투명한 근거 제공**: Black-box가 아닌 설명 가능한 가격 산정

---

## 6. 시스템 처리 흐름 개요

```
[입력]
사업계획서 (PDF, DOCX, 텍스트)
    ↓
[Process 1] 문서 분석 (Document Understanding)
    ↓
[Process 2] 프로젝트 구조화 (Ontology Mapping)
    ↓
[Process 3] 다중 전문가 분석 (Multi-Agent Reasoning)
    ├── 기술 전문가
    ├── 사업 전문가
    └── 경제 전문가
    ↓
[Process 4] 원가 계산 (Cost Engine)
    ↓
[Process 5] 전략적 가격 조정 (Pricing Engine)
    ↓
[출력]
최종 가격 + 산정 근거 + 리스크 설명
```

---

## 7. 성공 기준 (Success Criteria)

| 기준 | 목표 |
|------|------|
| 가격 산정 정확도 | 실제 계약 가격 대비 오차율 20% 이내 |
| 처리 속도 | 사업계획서 업로드 후 5분 이내 결과 제공 |
| 사용자 만족도 | 산정 근거의 이해 가능성 및 신뢰도 |
| 재사용성 | 유사 프로젝트 비교 분석 가능 |
