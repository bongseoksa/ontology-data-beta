### 포인트 1: [기업 대표/컨설팅] LLM 기반 '로직 주입형' 견적 시스템

- **핵심 문제 제기:** "LLM에게 그냥 물어보면 거짓말(Hallucination)을 합니다. 비즈니스 견적에 거짓말이 섞이면 곧 손실입니다."

- **차별화된 USP:** "우리는 모델을 믿지 않습니다. 우리는 **도메인 지식의 구조(Ontology)**를 설계하고, LLM은 이를 전달하는 인터페이스로만 활용합니다. 이것이 저희가 제공하는 **'안전한 AI 컨설팅'**의 핵심입니다."

### 포인트 2: [학생 멘토링] AI 에이전트의 '사고 복기' (Chain-of-Thought 시각화)

- **핵심 문제 제기:** "코딩은 이제 AI가 더 잘합니다. 여러분이 배워야 할 것은 '코딩'이 아니라 AI에게 일을 시키는 **'사고의 설계'**입니다."
- **시연 포인트:** Multi-Agent가 협업하여 복잡한 요구사항을 분석하고 스스로를 비판(Self-reflection)하는 과정의 시각화.
    - 결과물인 코드보다, AI가 **"왜 이 라이브러리를 선택했는가?"**, **"이 설계의 보안 취약점은 무엇인가?"**를 스스로 검토하는 로그를 보여줍니다.
- **차별화된 USP:** "저희 커뮤니티는 기술의 껍데기가 아닌 **본질(First Principles)**을 공부합니다. AI의 추론 과정을 설계할 수 있는 사람만이 AI 시대의 진짜 엔지니어가 될 수 있습니다."

## AI 프로젝트 견적 온톨로지 설계 (Data & Relationships)

### [데이터 엔티티 (Nodes)]

| **분류** | **데이터 항목 (구체적 변수)** | **유용한 외부 데이터 소스** |
| --- | --- | --- |
| **인프라** | GPU VRAM/TFLOPS, Instance hourly rate, Region, Egress 비용 | **AWS/GCP Pricing API**, **GetDeploying** (실시간 GPU 단가) |
| **모델** | Parameter count, Architecture type(Transformer, CNN 등), Precision(int4, fp16) | **Hugging Face Model Hub**, **Accelerate Estimate-memory** |
| **데이터** | Raw data size, Labeling complexity, Data drift frequency, Privacy level | **NIA AI 학습용 데이터 구축 가이드**, **K-DATA 데이터 품질 가이드** |
| **인력** | Role-based labor cost, Experience level, Domain expertise weight | **KOSA(한국소프트웨어산업협회) SW기술자 평균 임금** |
| **비즈니스** | Target Accuracy(%), Max Latency(ms), Concurrent users | **SPRI(소프트웨어정책연구소) AI 도입 대가 산정 보고서** |

### [관계 로직 (Relationships/Edges)]

1. **모델 파라미터($P$) $\xrightarrow{Impact}$ 인프라 사양($I$):**
    - $Memory_{required} \approx P \times Precision_{byte} \times 1.2$ (Buffer 포함). 이 관계를 통해 필요한 GPU 인스턴스 등급이 결정됩니다.
2. **데이터 품질($Q$) $\xrightarrow{Inversely\ Impact}$ 개발 공수($E$):**
    - 공공데이터(NIA)의 '데이터 정제 난이도' 지표를 활용하여, 초기 데이터 품질이 낮을수록 `Preprocessing_Task`의 가중치를 기하급수적으로 높입니다.
3. **목표 성능($A$) $\xrightarrow{Impact}$ 학습 반복 횟수($N$):**
    - 정확도 목표치가 95%에서 99%로 갈 때, 필요한 데이터 양과 학습 시간은 선형이 아닌 로그 함수 형태로 증가함을 온톨로지에 반영합니다.
4. **보안 등급($S$) $\xrightarrow{Impact}$ 아키텍처 복잡도($C$):**
    - 개인정보 포함 여부에 따라 `VPC`, `Anonymization Layer`, `On-premise` 노드가 활성화되며 전체 시스템 단가를 상승시킵니다.

## 3. 활용 가능한 구체적 외부 데이터셋 제안

이 데이터를 온톨로지에 주입(Ingestion)하면 견적의 객관성이 확보됩니다.

1. **KOSA 2025 SW사업 대가산정 가이드:**
    - **유용성:** AI 특화 '전문작업비' 및 '커스터마이징 비용'의 표준화된 항목(데이터형, 모델형)을 제공합니다.
    - **활용:** 기본 기능점수(FP) 외에 AI 특유의 비기능 요소를 SNAP(Software Non-functional Assessment Process) 점수로 환산하는 기준점으로 사용합니다.
2. **Hugging Face "Model Card" 데이터:**
    - **유용성:** 모델별 권장 하드웨어 사양과 학습에 소요된 Compute 자원 정보를 JSON으로 파싱 가능합니다.
    - **활용:** "이 정도 모델을 쓰려면 최소 A100 80GB가 몇 장 필요하다"는 근거를 생성합니다.
3. **NIA(한국지능정보사회진흥원) 구축 가이드:**
    - **유용성:** 원천데이터 수집 방식(크롤링, 촬영 등)에 따른 공정별 단가가 상세히 정의되어 있습니다.
    - **활용:** 중소기업 대표에게 데이터 구축 비용의 타당성을 설명할 때 가장 강력한 근거가 됩니다.
4. **GitHub Metrics (CodeState 등):**
    - **유용성:** 유사한 아키텍처의 오픈소스 프로젝트의 코드 복잡도와 이슈 해결 시간 데이터를 수집합니다.
    - **활용:** 특정 알고리즘 구현의 난이도를 '개발 속도' 데이터와 연결하여 현실적인 공기를 예측합니다.
