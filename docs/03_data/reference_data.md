## 7. 데이터 구축 계획

### 7.1 초기 구축 (MVP)

| 데이터 | 목표 건수 | 수집 방법 |
|--------|:--------:|----------|
| 프로젝트 사례 | 100건 | 수동 입력 |
| 리스크 사례 | 50건 | 수동 입력 |
| 산업 리포트 요약 | 20건 | 수동 요약 |

### 7.2 확장 계획

| 단계 | 데이터 규모 | 수집 방법 |
|------|------------|----------|
| 1단계 | 500건 | 반자동 (크롤링 + 검수) |
| 2단계 | 2,000건 | 자동 + 품질 관리 |
| 3단계 | 10,000건+ | 커뮤니티 기여 포함 |

### 7.3 데이터 품질 관리

```
[품질 기준]
├── 완전성: 필수 필드 100% 입력
├── 정확성: 검증된 데이터만 등록
├── 일관성: 표준 스키마 준수
├── 최신성: 3년 이내 데이터 우선
└── 관련성: 도메인 범위 내 데이터

[품질 관리 프로세스]
1. 데이터 수집
2. 형식 검증
3. 내용 검토
4. 분류/태깅
5. 등록 승인
```

## 8. RAG 인덱싱 전략

### 8.1 청킹 전략

| 데이터 유형 | 청킹 방식 | 크기 |
|------------|----------|------|
| 프로젝트 사례 | 문서 단위 | 1건 = 1청크 |
| 리포트 | 섹션 단위 | 500~1000자 |
| 가이드 | 주제 단위 | 500~1000자 |

### 8.2 메타데이터 설계

```json
{
  "doc_id": "unique_id",
  "doc_type": "case/report/guide/risk",
  "industry": ["공공", "금융"],
  "project_type": ["SI", "웹"],
  "size_category": "중형",
  "date": "2024-01",
  "source": "napa.kr",
  "reliability": "high/medium/low"
}
```

### 8.3 검색 시나리오

| 쿼리 유형 | 검색 대상 | 예시 |
